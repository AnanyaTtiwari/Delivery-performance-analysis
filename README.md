# Delivery Performance Analysis Dashboard

## Project Overview
Efficient delivery operations are critical for customer satisfaction and business success in e-commerce and quick-commerce industries. Delayed deliveries, poor service ratings, and refund requests can directly impact customer trust and operational efficiency.

This project focuses on analyzing delivery performance across multiple platforms to identify inefficiencies, delay patterns, refund behavior, and customer satisfaction trends.

The project was built using **Python, MySQL, and Power BI** to perform end-to-end data cleaning, analysis, and dashboard visualization.

---

## Problem Statement
Delivery platforms often face challenges such as late deliveries, poor customer ratings, and increasing refund requests. Businesses need a data-driven system to monitor performance and optimize operations.

This project helps answer:

- Which platform performs best?
- What is the average delivery time?
- Which categories take longer to deliver?
- How do delays impact customer ratings?
- What is the relationship between delays and refunds?

---

## Objectives
The main objectives of this project are:

- To analyze platform-wise delivery performance
- To calculate average delivery time
- To identify late delivery percentage
- To study customer rating patterns
- To analyze refund trends
- To compare product category performance

- To generate business recommendations

- Service Rating

## Key Insights

- Some platforms showed significantly better delivery efficiency.
- Late deliveries negatively affected customer ratings.
- Higher delivery delays increased refund requests.
- Certain product categories took longer to deliver.
- High-value orders often required more delivery time.
- Platform efficiency varied significantly.
